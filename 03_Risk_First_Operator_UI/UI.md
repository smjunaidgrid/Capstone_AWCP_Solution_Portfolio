# 03 Risk-First Operator UI

This folder contains the React-based frontend dashboard for the Agent Workforce Control Plane (AWCP). It acts as the "Human Control Room," providing a risk-first view of autonomous agent activity, degraded workflows, and real-time governance telemetry.

## Domain Purpose

The Operator UI domain focuses on:
- Providing real-time visibility into agent execution across the enterprise.
- Highlighting degraded workflows and budget breaches immediately.
- Displaying live Replay Summaries generated by the Data Science Cognitive Shield.
- Monitoring the health of the Intake Proxy and Evidence Ledger.

## Core Architecture Components

* **Live Control Surface:** A polling dashboard that fetches the latest system state and telemetry from the FastAPI backend every 3 seconds.
* **KPI Metrics Grid:** Tracks critical numbers such as Workflows Degraded, Active Proxies, and Actions Governed.
* **Cognitive Shield Live Feed:** Directly displays the human-readable `latest_assessment` to explain *why* an agent succeeded or failed.
* **Workflow Queue & Action Gates:** Displays the exact UI layout required for operators to prioritize workflows by failure budgets and visualize state-changing actions.

## Data Flow & Integration

The UI is fully decoupled from the execution engines. It relies strictly on the `GET /status` endpoint exposed by the Workflow Intake Proxy (`intake_proxy.py`). 

When the backend processes a task (via Temporal and the Python Worker) and evaluates it (via the Cognitive Shield), the resulting assessment is published to the central system state. The React UI polls this state, ensuring the human operator always sees the most up-to-date execution evidence.
